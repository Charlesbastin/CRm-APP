## 🎯 Overview

TechnoCraft Event Portal is a **role-based CRM and event management application** designed for technical fests, college events, and institutional programs. It runs as a **WebView-based Android app** powered by a vanilla HTML/CSS/JS frontend and Firebase backend.

The app supports the full lifecycle of event management:
> Registration → Payment (UPI) → Approval → QR Pass → Attendance Scan → Certificate

---

## ✨ Features

### 👤 Authentication & Roles
- Firebase Email/Password authentication
- Role-based access control (Student / Coordinator / Admin)
- Coordinators require Admin approval before activation
- Admin accounts can only be created server-side (never from the app)

### 📅 Event Management
- Create, edit, and manage events with capacity, entry fee, and date
- Active Coordinators and Admins can create events
- Event listing with search and filter

### 📝 Registrations
- Students can register for events
- Registration status: `Pending → Approved / Rejected`
- Coordinators can approve/reject registrations for their own events
- Students cannot self-approve (enforced at Firestore rules level)

### 💳 Payments (UPI)
- Integrated UPI payment flow
- Payment records written only by Cloud Functions (server-side, tamper-proof)
- Students can view their own payment history

### 📷 QR Code System
- QR passes generated by Cloud Functions after payment confirmation
- Live QR scanner for Coordinators to mark attendance
- Uses `html5-qrcode` library for real-time camera scanning

### 🏆 Certificates
- Digital certificates auto-generated by Cloud Functions post-attendance
- Students can view and download their certificates

### 🔔 Notifications
- In-app notification system
- Staff can send notifications to any user
- Students can mark notifications as read

### 📊 Admin Dashboard
- Full user management (promote/demote roles, activate accounts)
- Audit logs (immutable, server-side only)
- Global settings management
- View all payments, registrations, and feedback

## 👥 User Roles

| Role | Capabilities |
|------|-------------|
| **Student** | Register for events, make payments, view QR pass, track attendance, download certificates |
| **Coordinator** | Create events, approve/reject registrations, scan QR attendance, view event feedback |
| **Admin** | Full access — manage users, events, payments, audit logs, global settings |

> ⚠️ **Role Security**: Roles are always read from Firestore server-side. Users cannot self-assign Admin or Coordinator roles.

## 🚀 Getting Started

### Prerequisites

- [Node.js](https://nodejs.org/) v18+
- [Firebase CLI](https://firebase.google.com/docs/cli): `npm install -g firebase-tools`
- [Android Studio](https://developer.android.com/studio) (for APK builds)
- A Firebase project with **Authentication**, **Firestore**, and **Cloud Functions** enabled

### 1. Clone the Repository

```bash
git clone https://github.com/Charlesbastin/CRm-APP.git
cd CRm-APP
```

### 2. Run Locally (Web)

Simply open `index.html` in a browser, or use a local server:

```bash
npx serve .
```

> Configure `firebase-config.js` with your own Firebase project credentials first.

## 🔥 Firebase Setup

### 1. Create Firebase Project

1. Go to [Firebase Console](https://console.firebase.google.com/)
2. Create a new project
3. Enable **Authentication** (Email/Password provider)
4. Enable **Firestore Database**
5. Enable **Cloud Functions**

### 2. Configure Firebase

Edit `firebase-config.js` with your project credentials:

```js
window.TECHNOCRAFT_FIREBASE_CONFIG = {
  apiKey: "YOUR_API_KEY",
  authDomain: "YOUR_PROJECT.firebaseapp.com",
  projectId: "YOUR_PROJECT_ID",
  storageBucket: "YOUR_PROJECT.appspot.com",
  messagingSenderId: "YOUR_SENDER_ID",
  appId: "YOUR_APP_ID"
};
window.TECHNOCRAFT_FIREBASE_ENABLED = true;
```

### 3. Deploy Firestore Rules

```bash
firebase login
firebase use --add
firebase deploy --only firestore:rules
```

### 4. Deploy Cloud Functions

```bash
cd firebase/functions
npm install
cd ../..
firebase deploy --only functions
```

### 5. Bootstrap Admin Account

After deploying, run the admin bootstrap script once to create the first Admin user:

```bash
node firebase/admin-bootstrap.js
```

## 🔒 Security Model

The app uses a **defense-in-depth** security approach:

1. **Authentication**: Firebase Auth validates every user session
2. **Firestore Rules**: Server-side rules enforce all role-based access (cannot be bypassed from the client)
3. **Role Immutability**: Users cannot change their own role or email via the app
4. **Payment Integrity**: All payment records are written only by Cloud Functions using the Admin SDK
5. **Attendance Immutability**: Attendance records cannot be updated or deleted once created
6. **Audit Trail**: All admin actions are logged (server-side, immutable)

## 🗃 Firestore Collections

| Collection | Purpose | Write Access |
|-----------|---------|-------------|
| `users` | User profiles and roles | Self (limited) + Admin |
| `events` | Event details | Coordinators + Admin |
| `registrations` | Student event registrations | Students + Coordinators + Admin |
| `payments` | Payment records | Cloud Functions only |
| `passes` | QR pass tokens | Cloud Functions only |
| `attendance` | Attendance log (immutable) | Coordinators + Admin |
| `certificates` | Digital certificates | Cloud Functions only |
| `notifications` | In-app notifications | Staff |
| `feedback` | Event feedback from students | Students |
| `auditLogs` | Admin action audit trail | Cloud Functions only |
| `settings` | Global app settings | Admin only |
